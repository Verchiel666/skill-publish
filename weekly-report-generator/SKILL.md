# 行政周报生成 Skill

## 概述

专为行政/文秘/基层管理者设计的结构化工作汇报生成器。把一周零散的工作记录自动转化为格式规范、逻辑清晰、可直接提交的工作报告。

**V3.0 新增**：时间轴视图、自定义指标面板、事项交叉关联、表达多样化引擎、参考模板文件、自动保存机制。

---

## 任务公式

> 通过**结构化提炼 + 数据汇总 + 进度追踪 + 计划对照 + 表达多样化 + 模板匹配**，为**行政/文秘/基层管理者**完成**从零散记录到可提交/可群发/可存档的全场景工作汇报**。

## 标准报告结构（详尽模式）

```
[时间段]工作汇报

一、本周摘要（2-3句核心概况）

二、重点事项时间轴（新增：按日排列本周里程碑）
  周一：[关键事项]
  周二：[关键事项]
  ...

三、重点工作
（一）[板块] — 关联：[关联的其他事项]
  1. [事项] [█████ 100%] — 进度/结果
  2. [事项] [███░░ 60%] — 进度/阶段
（二）[板块]

四、上周计划完成情况
| 上周计划 | 状态 | 偏差说明 | 后续 |
|----------|------|---------|------|
| XXX | ✅ | — | — |
| YYY | 🔄 | 延后2天 | 下周一前 |
| ZZZ | ❌ | 等审批 | 周三跟进 |

五、数据面板
[使用自定义指标或默认面板]

六、问题与建议
（一）[问题] — [影响] — [建议]
（二）[问题]

七、下周计划
（一）[高优先级] ⭐
（二）[常规] 
（三）[承接本周未完成]

[汇报人/部门]
[日期]
```

## V3.0 新增功能

### 时间轴视图

详尽模式下自动生成按日排列的里程碑时间轴，方便领导快速扫读"这一周发生了什么"。

**原则**：
- 每个工作日一个条目
- 无关键事项的日期标注"常规运营"
- 同一日多个事项→取最重要的 1-2 个
- 时间轴上不重复列举已在重点工作区详述的内容，只列标题

### 自定义指标面板

用户可以设定本单位关注的 KPI，后续周报自动按自定义面板呈现。

**默认面板**（行政通用）：
```
文件处理 X件 | 会议组织 X场 | 检查接待 X次 | 设施维护 X项
```

**自定义示例**：
- "我们主要看文件量和接待量" → 面板重点突出这两项
- "加上培训人次和满意度" → 在面板中增加指标
- 设定后，会话内所有后续报告自动沿用自定义面板

### 事项交叉关联

当多个事项存在因果/协同关系时，自动标注关联，避免"孤立罗列"。

**规则**：
- 因 A 导致的 B → "B（关联：承接 A 结果）"
- A 和 B 同属一个大项目 → 归为同一板块，加项目总进展
- 会议决策→后续行动 → 自动链接

### 表达多样化引擎

避免周报"千篇一律"被领导觉得敷衍。

**变体策略**：
- 摘要句法轮换：每次使用不同句式（本周重点推进了… / 过去一周聚焦于… / 本周围绕…开展工作）
- 动词库：完成/推进/启动/优化/协调/落实/组织/处理/完成等
- 同一事项连续 3 周出现 → 自动更换表述角度（从"做了什么"到"进展到哪"到"下一步要破什么"）
- 周报中的"本周无异常"→轮换为"运转平稳""各项指标正常""按计划推进中"

## 执行规则

1. **拒绝编造**：没给的数字标「待补充」
2. **自动归类**：散乱信息智能识别板块
3. **计划对照**：有上周报告时自动核对完成率
4. **进度可视化**：长周期事项加进度条
5. **遗留承接**：本周未完成→自动写入下周
6. **优先级排序**：领导交代 > 对外 > 紧急 > 常规
7. **问题有分寸**：客观陈述+建议方案
8. **表达不重复**：每周轮换句式（V3新增）
9. **生成后自动保存**：方便下次对比（V3新增）

## 智能特征

### 工作自动归类
| 关键词 | 板块 | emoji |
|--------|------|-------|
| 文件/文书/印章/收发/传阅/归档 | 文书管理 | 📁 |
| 会议/活动/接待/迎检/调研 | 会议活动 | 📋 |
| 采购/维修/报修/物业/食堂 | 后勤保障 | 🔧 |
| 人事/考勤/请假/入职/离职 | 人事管理 | 👥 |
| 联络/协调/发函/通知 | 对外协调 | 📞 |
| 档案/数字化/系统/OA | 信息化 | 💻 |
| 培训/学习/考试/演练 | 培训演练 | 🎓 |
| 财务/预算/报销/审计 | 财务管理 | 💰 |
| 安全/消防/防疫/应急 | 安全管理 | 🛡️ |
| 党建/学习/组织生活 | 党建群团 | 🚩 |

**不匹配时**：归入"其他事务"，不强行归类。

### 进度自动判断
| 用户表述 | 状态 | 进度条 |
|---------|------|--------|
| "做完了""已完成""搞定了" | ✅ | `█████ 100%` |
| "推进中""做了一半""还在搞" | 🔄 | `███░░ 60%` |
| "刚开始""启动" | 🔄 | `█░░░░ 20%` |
| "还没开始""等审批" | ❌ | 无条，标注原因 |
| "卡住了""遇到问题" | ⚠️ | 标注+纳入问题区 |

### 汇报对象自适应
- **直属领导**：简练直接，重点"做了什么+结果"
- **大领导/对外**：多说"价值+影响"，少说过程
- **全部门/全员**：注重"协同+配合"

## 边缘预案

| 场景 | 处理 |
|------|------|
| "没什么特别的" | 列常规+轮换表述"本周运转平稳" |
| 只给了开会 | 最多 2 问 |
| 散乱口述夹抱怨 | 客观化，去情绪 |
| 无数据 | 描述不编造 |
| 上周数据忘了 | 不追问 |
| "抄上周格式" | 搜会话历史 |
| 内容太少 | 不注水，精简易读 |
| 多项目并行 | 独立进度条 |
| "上次说的事呢" | 历史计划逐项核对 |
| 发企业微信群 | 切 IM 即贴 |
| **自定义指标** | 设定后自动沿用 |
| **连续 3 周相似** | 表达轮换，防流水账 |
