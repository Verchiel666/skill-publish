# 行政公文起草 Skill

## 概述

专为行政/文秘/办公室人员设计，覆盖通知、请示、函件、纪要等 15+ 种公文类型的**起草→改写→批量→修正**全流程。支持三档模式、Word 即贴格式、对话式修改、多人发言纪要、批量生成、类型歧义澄清、改写现有草稿。

**V5.0 新增**：改写模式（草稿→正式公文）、批量生成（同模板多抬头）、纠错指令、类型歧义澄清、档位冲突自动判定、沉默确认机制。

---

## 任务公式

> 通过**模板匹配 + 格式化生成 + 改写 + 批量 + Word 即贴输出**，为**行政/文秘/基层管理者**完成**从起草到改写到批量的公文全流程处理**。

## 正向触发器

- "写个/起草/帮我拟/这个公文怎么写 + [公文关键词]"
- "帮我改写/润色/正式化 + [一段草稿]"
- "通知/请示/函件/纪要/报告/通报/批复/讲话稿/致辞/计划/总结/制度"
- "速稿""快速出" → 精简模式
- "红头""正式文件""对外发" → 详尽模式

---

## 反向过滤器

以下场景**不触发**：
- 私人信件、微信消息、朋友圈文案
- 合同审查/法律文书
- 技术文档/API 文档/代码注释
- 学术论文/毕业论文
- 营销文案/广告词/推广文章

## 新增：改写模式

用户提供一段现有文本要求"改成正式通知/公文"时，**正常触发**。这是核心场景之一——行政人员经常自己写了草稿需要正式化。

判断标准：用户给了具体文本 + "改写成/润色成/正式化成 + 公文关键词" = 触发改写模式。

## 输出标准

### Word 即贴格式

生成公文输出**纯文本格式**（无 Markdown 符号），复制全选→粘贴到 WPS/Word 无需二次排版。输出末尾提示：

> 📋 以上内容可直接全选复制，粘贴到 WPS/Word 后无需二次排版。

## 三档模式 + 冲突判定

| 模式 | 字数 | 触发词 | 适用 |
|------|------|--------|------|
| 🚀 精简 | 100-200字 | "速稿""群里发""简单写" | 内部群通知 |
| 📋 标准 | 200-800字 | 默认 | 日常工作 |
| 📄 详尽 | 500-2000字 | "正式""对外""红头" | 对外/上级 |

**档位冲突自动判定**：当用户同时给出冲突信号时（如"速稿，但要正式"），按以下优先级裁决：
1. **接收方优先级最高**：对外/上级 → 强制详尽模式
2. **格式词 > 速度词**："红头发给市里" → "市里"触发对外→详尽，忽略"速"
3. **精致 > 速度**："快速但正式" → 冲突时正式优先
4. **可以降半档**：对外通知可以用标准档，不需要强制详尽

输出时明确告知用户："采用了 XXX 模式（因为接收方/格式要求优先于速度要求）"

## 批量生成模式

当用户说"给每个部门发一个""通知各科室/各部门""全部改抬头"时，触发批量模式：

**流程**：
1. 确认通用模板内容
2. 确认需要生成的抬头列表（如不清楚，问：需要发给哪些部门？）
3. 用同一模板分别填入不同抬头，连续输出

**格式**：
```
--- [部门A] ---
[公文内容，抬头为部门A]

--- [部门B] ---
[公文内容，抬头为部门B]
```

**规则**：
- 同模板不同抬头，内容一致
- 每个部门之间用分隔线隔开
- 落款用发送方统一单位

## 类型歧义澄清

当用户用词可能对应多种公文类型时，**主动确认**：

| 常见歧义 | 可能类型 | 确认方式 |
|---------|---------|---------|
| "写个报告" | 报告/汇报/总结 | "汇报工作还是汇报一件事情？" |
| "通知他们" | 通知/函件/通告 | "是发给内部还是外部？" |
| "通报一下" | 通报/通知 | "是传达信息还是表彰/批评？" |
| "发个文件" | 全类型 | "大概什么内容？" |
| "出个文" | 全类型 | "对上级还是对内部？" |

## 质量红线

### ✅ 好输出
- 公文类型正确 | 标题格式规范 | 正文逻辑清晰分层
- 主送+落款+日期齐全 | 用语正式规范 | 无编造信息
- Word 即贴格式

### ❌ 差输出
- 类型不匹配 | 落款/主送缺失 | 一段到底
- 编造信息 | 口语化 | 含 Markdown 符号

## 边缘预案

| 场景 | 处理 |
|------|------|
| 只说"写个通知" | 最多 2 问：什么事？发给谁？ |
| 信息不全 | 标注「请补充」，其余正常生成 |
| 语气/档位调整 | 说出即可，如"改正式一点" |
| 不确定类型 | 给 1-2 选项，确认后生成 |
| "按以往格式" | 先搜会话历史输出，无则要求范例 |
| 散乱信息 | 先梳理要点清单，确认后生成 |
| 敏感内容 | 正常生成 + "请做好内部合规审查" |
| 错别字输入 | 自动纠正同音错字 |
| **改写草稿** | 保留用户全部事实信息，只改格式和用语 |
| **批量多抬头** | 确认模板→确认抬头列表→连续输出 |
| **用户改主意** | 支持纠错指令（见对话修改指令） |
| **附件提醒** | 用户提到附件/清单但未给内容时，输出末尾标注：「附件：[请补充附件名称及内容]」 |
| **输出后无反馈** | 输出后加一句："需要调整格式、语气或补充内容吗？随时说。" |

## 对话式修改指令

| 指令 | 效果 |
|------|------|
| "改正式一点" | 升一档 |
| "太长了，缩一下" | 降至精简模式 |
| "加一段XXX" | 正文插入新段落 |
| "语气改委婉" | 降低措辞硬度 |
| **"时间改成XXX"** | **单改时间/地点/人名，正文不动** |
| **"不是通知，是请示"** | **纠正公文类型，重新生成** |
| **"落款去掉/换成XX"** | **单改落款** |
| "撤回到上一版" | 恢复上次输出 |
| "抄送我一份" | 正文尾部加"抄送：XX" |
| "要红头格式" | 加红头行+文号 |
| "就按这个发吧" | 输出最终版，移除「请补充」标注 |
| "直接给 Word 版" | 输出 Word 即贴格式（默认） |

## 沉默确认

每次输出后，根据情况加一句确认引导：
- 有「请补充」标注 → "以上内容需要补充的地方随时告诉我。"
- 无「请补充」→ "需要调整格式或内容吗？"
- 批量模式末尾 → "以上 N 份，需要调整哪一份？"

**原则**：只问一句，不连环追问。

---

## 会议纪要：多人发言智能区分

多人发言散乱口述时，按发言人整理：

格式：
```
一、[发言人]：[发言主旨]
二、[发言人]：[发言主旨]
```

原则：
- 同一人连续发言合并
- 去掉口语语气词
- 保留关键数据和结论
- 无法确定发言人时用"某负责同志"，不编造

## 执行工作流

### 第一步：判断模式
- **起草模式**（默认）：从零生成公文
- **改写模式**：用户给了草稿文本 → 转第二步改写
- **批量模式**：多抬头 → 转批量流程

### 第二步：识别公文类型
自动匹配 15 种类型。有歧义时主动确认。

### 第三步：收集信息 + 套模板 + 判档位
- 选模板框架
- 收集缺失信息（最多 2 问）
- 会话记忆复用落款/部门名
- 档位冲突时自动判定并告知

### 第四步：生成公文（Word 即贴格式）

### 第五步：自检
类型→标题→主送→逻辑→落款→信息真实→格式，逐项过。

### 第六步：沉默确认
加一句确认引导，等待反馈。

## 完整模板库

### 通知
```
关于[事项]的通知

[接收方]：

[背景，可选]现将有关事项通知如下：

一、[要点]
二、[要点]

[要求]

特此通知。

[落款]
[日期]
```

### 请示
```
关于[事项]的请示

[上级]：

[背景+理由]

一、[方案]
二、[必要性]

妥否，请批示。

[落款]
[日期]
```

### 纪要
```
[会议名称]会议纪要

一、会议基本情况
时  间：[   ]
地  点：[   ]
主持人：[   ]
参会人：[   ]
记录人：[   ]

二、会议主要内容
（一）[议题]
（二）[议题]

三、议定事项
（一）[事项]
（二）[事项]

[落款]
[日期]
```

### 函件
```
关于[事宜]的函

[接收方]：

[事由]

[请求或说明]

感谢支持。

[落款]
[日期]
（联系人：[   ]  联系电话：[   ]）
```

### 讲话稿/致辞
```
[标题]

各位[称呼]：

[场景引入]

[主体：3-4 个要点]

[结尾：号召/祝愿/总结]

[署名]
[日期]
```

### 工作总结
```
[时间段]工作总结

一、工作完成情况
（一）[方面]：[简述+数据]
（二）[方面]：[简述+数据]

二、存在问题
（一）[问题]
（二）[问题]

三、下一步工作计划
（一）[计划]
（二）[计划]

[落款]
[日期]
```

### 制度/办法
```
[单位][事项]管理办法

第一条  为[目的]，根据[依据]，制定本办法。
第二条  [适用范围]
第三条  [主要原则]
...
第X条  本办法自[日期]起施行。

[落款]
[日期]
```

## 执行规则

1. **绝对禁止编造**
2. **Word 即贴格式**（无 Markdown 符号）
3. **一次最多问 2 个问题**
4. **主动推断合理内容**
5. **落款日期默认当天**
6. **会话内记忆落款**
7. **自动纠正同音错字**
8. **纪要区分发言人**
9. **档位冲突时正式优先**
10. **类型歧义主动确认**
